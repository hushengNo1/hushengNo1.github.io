# 2.使用gzip压缩

参考：

nginx和vue开启双gzip压缩的方法 (opens new window)

nginx中如何设置gzip（总结） (opens new window)

vue 项目部署 nginx 开启gzip优化性能 (opens new window)

vue.config.js配置：

const CompressionPlugin = require("compression-webpack-plugin");
module.exports = {
  configureWebpack: {
    plugins: [
      new CompressionPlugin({
        filename: "[path].gz[query]",
        algorithm: "gzip",
        test: new RegExp("\\.(js|css)$"),
        // 只处理大于xx字节 的文件，默认：0
        threshold: 10240,
        // 示例：一个1024b大小的文件，压缩后大小为768b，minRatio : 0.75
        minRatio: 0.8, // 默认: 0.8
        // 是否删除源文件，默认: false，不能删除，因为HTML文件中引用的还是js文件
        deleteOriginalAssets: false
      })
    ]
  }
};

# 琐碎知识点

  1. 过滤器调用取就近原则（全局过滤器和私有过滤器重名时取私有过滤器）；

  2. 安装@vue/cli@3.9.2时权限不够，添加参数 --unsafe-perm，即：sudo npm install -g @vue/cli --unsafe-perm

  3. vue-cli 3.x版本全局环境变量使用 (opens new window)

  4. 通过v-for循环生成的ref是一个数组，需要这样取：

    • this.$refs[refName][0]
      1

  5. 使用keep-alive缓存页面时，如果页面中数据需要改变，用activated生命周期钩子

  6. vue项目总css使用webpack路径别名

    • 参考 (opens new window)

    • JS可以用alias，css就报错

      //js中:
	import tool from '@/utils/xxx' (可以引用)
//css中:
	@import '@/assets/css/reset.sass' (报错)
      1
      2
      3
      4

      解决： 在引用路径的字符串最前面添加上 ~ 符号，如 @import “~@/style/theme”；Webpack 会将以 ~ 符号作为前缀的路径视作依赖模块而去解析，这样 @ 的 alias 配置就能生效了。

  7. 递归组件：组件自身调用自身；当定义递归组件时，name属性必须加上。

  8. v-on,@的值可以是对象

  9. 创建和编辑的页面使用的是同一个component,默认情况下当这两个页面切换时并不会触发vue的created或者mounted钩子，官方说你可以通过watch $route的变化来做处理，但其实说真的还是蛮麻烦的。后来发现其实可以简单的在 router-view上加上一个唯一的key，来保证路由切换时都会重新渲染触发钩子了。

    <router-view :key="key"></router-view>
computed: {
    key() {
        return this.$route.name !== undefined? this.$route.name + +new Date(): this.$route + +new Date()
    }
 }
